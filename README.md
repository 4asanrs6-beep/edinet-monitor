# EDINET Monitor

EDINET（金融庁 電子開示システム）の新着開示をリアルタイム監視するローカルデスクトップアプリ。

日本株の短期売買支援を目的に、需給インパクトの大きい開示（大量保有報告書、変更報告書、増資、CB、第三者割当など）を即座に検知・通知する。

## 機能

- EDINET API v2 を定期ポーリングし新着開示を自動検出
- 大量保有報告書・変更報告書・増資・CB・MSワラント・第三者割当・自己株取得・TOB・主要株主異動を優先検出
- Windows デスクトップ通知（トースト通知）
- 書類の優先度自動判定・色分け表示
- 未読/既読管理
- イベント種別フィルタ・銘柄検索
- メモ機能
- PDF直接表示
- ウォッチリスト銘柄の優先度引き上げ
- システムトレイ常駐
- SQLite によるログ永続化

## 必要環境

- Python 3.10 以上
- Windows 10 / 11
- EDINET API サブスクリプションキー

## セットアップ

### 1. 依存パッケージのインストール

```bash
cd edinet_monitor
pip install -r requirements.txt
```

### 2. EDINET APIキーの取得

1. [EDINET API](https://disclosure.edinet-fsa.go.jp/) にアクセス
2. ユーザー登録し、APIサブスクリプションキーを取得
3. `config.yaml` の `edinet.api_key` にキーを設定

### 3. 起動

```bash
python main.py
```

## 設定 (config.yaml)

| 項目 | 説明 | デフォルト |
|------|------|-----------|
| `edinet.api_key` | EDINET API キー | （必須） |
| `edinet.polling_interval_sec` | ポーリング間隔（秒） | 60 |
| `edinet.max_retries` | エラー時リトライ回数 | 3 |
| `monitoring.enabled_categories` | 監視対象カテゴリ | 全主要カテゴリ |
| `monitoring.watchlist_sec_codes` | ウォッチリスト銘柄コード | [] |
| `monitoring.skip_corrections` | 訂正書類をスキップ | true |
| `notification.enabled` | デスクトップ通知 | true |
| `notification.sound` | 通知音 | true |

### ウォッチリスト設定例

```yaml
monitoring:
  watchlist_sec_codes:
    - "7203"  # トヨタ
    - "9984"  # ソフトバンクG
```

ウォッチリストに登録した銘柄の開示は優先度が1段階引き上げられる。

## 使い方

### 画面構成

- **上部ツールバー**: 種別フィルタ、検索、未読/全て切替、手動取得、監視ON/OFF
- **左パネル**: イベント一覧（優先度順・時刻順）
- **右パネル**: 選択した書類の詳細表示、メモ、PDFダウンロード
- **下部ステータスバー**: 監視状態、最終取得時刻、未読件数

### 操作

- **書類選択**: 左パネルで書類をクリック → 右パネルに詳細表示
- **既読/未読切替**: ダブルクリック or 詳細パネルのボタン
- **PDF表示**: 詳細パネルの「PDF表示」ボタン（自動ダウンロード＆表示）
- **メモ保存**: 詳細パネル下部のメモ欄に記入 →「メモ保存」ボタン
- **手動取得**: ツールバーの「手動取得」ボタン
- **ウィンドウ閉じる**: システムトレイに最小化（pystray導入時）

### 優先度の色分け

| 優先度 | 色 | 対象 |
|--------|-----|------|
| 最重要 | 赤 | 大量保有報告書、変更報告書、TOB、主要株主異動 |
| 重要 | オレンジ | 増資、CB、MSワラント、第三者割当、自己株取得 |
| 中 | 青 | 臨時報告書 |
| 低 | グレー | その他 |

## イベント分類ロジック

書類名（`docDescription`）のキーワードマッチを主軸に分類。`docTypeCode` は補助的に使用。

### 分類ルール（優先順）

1. 変更報告書 → キーワード「変更報告書」
2. 大量保有報告書 → キーワード「大量保有報告書」（「変更」を除外）
3. 公開買付 → キーワード「公開買付届出書」「公開買付報告書」等
4. 主要株主異動 → キーワード「主要株主の異動」
5. MSワラント → キーワード「行使価額修正条項付」「MSワラント」等
6. CB/転換社債 → キーワード「新株予約権付社債」「転換社債」等
7. 第三者割当 → キーワード「第三者割当」
8. 増資 → キーワード「有価証券届出書」「発行登録追補書類」
9. 自己株取得 → キーワード「自己株券買付状況報告書」等
10. 臨時報告書 → キーワード「臨時報告書」

訂正書類（書類名に「訂正」を含む）はデフォルトでスキップ（設定変更可）。

## 設計上の仮置き・制限事項

- **XBRL解析**: MVP版では未実装。保有比率・希薄化率等の詳細情報は書類PDFから確認。将来的にXBRLパーサーを追加予定。
- **docTypeCode**: EDINET API v2 の書類種別コードは、書類名キーワードマッチを主軸とし補助的に使用。APIの仕様変更に強い設計。
- **増資の細分類**: 有価証券届出書は広範な書類を含むため、MVP版では「増資」として一括分類。将来的に府令コード・様式コードで細分化予定。
- **ポーリング間隔**: デフォルト60秒。EDINET APIの利用規約・レートリミットに準拠すること。重要時間帯（8:45-9:00, 15:00-15:30等）のみ短縮する拡張を将来予定。
- **対象日**: 当日分のみ監視。過去日の遡及取得は手動取得で対応。
